## 📝 问题详情

### 001-自动化测试无法执行

**问题现象**：
- 测试在 Test Automation 窗口中能被识别（显示测试列表）
- 但运行后显示 "0次测试/0次失败/0秒"
- 测试代码实际上没有被执行

**根本原因**：
1. 使用了错误的 `EAutomationTestFlags::ApplicationContextMask` 标志
2. 缺少 `WITH_DEV_AUTOMATION_TESTS` 宏保护

**修复方案**：
- 将 `ApplicationContextMask` 替换为 `EditorContext`
- 添加条件编译宏保护
- 优化 Build.cs 配置

**详细文档**：[001-AutomationTestNotRunning.md](./001-AutomationTestNotRunning.md)

**发现日期**：2025-11-30  
**影响范围**：所有自动化测试  
**修复提交**：`2b5c844`

## 🤝 贡献指南

### 添加新问题

1. 确保问题具有**代表性**和**参考价值**
2. 问题必须是**经过多次尝试才发现**的非常规问题
3. 必须包含**完整的修复方案**和**验证步骤**
4. 代码示例要**清晰**、**可复现**

- ✅ 问题描述清晰，能让读者快速理解
- ✅ 根本原因分析透彻，有理论依据
- ✅ 修复方案可操作，步骤详细
- ✅ 包含代码示例和截图（如有必要）
- ✅ 标注严重性、修复难度、影响范围
